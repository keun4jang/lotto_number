# 크라우드(구매 편향) 모델 실측 보정 보고서 — be-v1.3.0-ev

> 통계 기반 추천, EV 관점 비인기 조합 회피, **당첨 보장 없음**.
> 이 작업은 당첨 확률을 바꾸지 않는다(불가능). 개선하는 것은 오직
> "사람들이 어떤 조합을 많이 사는가"에 대한 추정, 즉 당첨 시 기대
> 공동 당첨자 수(상금 분할) 추정이다.

## 1. 방법론

한국 로또 1등은 패리뮤추얼 방식이므로, 회차별 **1등 당첨자 수**는 그
회차 당첨 조합이 실제 한국 구매자들 사이에서 얼마나 인기 있었는지에
대한 직접 측정치다.

- 균등 무작위 구매 가정 시 기대 1등 당첨자 수:
  `expected = (total_sales / 1,000원) / 8,145,060`
- 실측 인기도: `popularity_ratio = first_winners / expected`
- 회귀 목표변수: `log((observed + 0.5) / expected)` (연속성 보정)
- 가중치: `expected` (포아송 분산 근사 → WLS)

**표본**: 106~1229회차 중 expected ≥ 1 인 1,128회차.
105회차 이전(게임당 2,000원 시대)은 환산 기준이 달라 제외.

**우아한 성질**: 당첨 조합은 조합 공간에서 균등 무작위로 추첨되므로,
"당첨 조합만 관측한다"는 사실이 선택 편향을 만들지 않는다 — 이 표본은
인기도 함수에 대한 **비편향 표본**이다.

구현: `src/lotto_doctor/crowd_calibration.py` (순수 함수 API).

## 2. 피처별 발견 (한국 실측 데이터 기준)

간명 모형(WLS, N=1,128) 결과:

| 피처 | 계수(log-ratio) | p값 | 판정 |
|---|---|---|---|
| 1~12 번호 개수 (생일 '월') | +0.049 /개 | 0.0009 | **확인** — 과선택 |
| 등차수열 점수 | +0.71 | <0.0001 | **강하게 확인** — 규칙 간격 과선택 |
| 13~31 vs 32~45 | −0.019 /개 | 0.14 | 유의하지 않음 (생일 '일' 구간 우위 미확인) |
| 7 포함 | +0.028 | 0.45 | 방향만 양(+), 유의하지 않음 |
| 7의 배수(7 제외) | −0.028 | 0.11 | 유의하지 않음 |
| 연속 번호 쌍 | −0.026 | 0.23 | 유의하지 않음 |
| 용지 행/열/대각선, 클러스터 | ≈0 | >0.5 | 유의하지 않음 |
| '균형 잡힌' 조합(pretty) | +0.062 | 0.27 | 약한 양의 방향 |
| 직전 회차 겹침 | −0.09 | 0.14 | 유의하지 않음 (오히려 음의 방향) |
| 4 기피 / 13 기피 | +0.006 / +0.053 | 0.87 / 0.13 | **미확인** (기피 증거 없음) |

핵심 해석:
- **자동/반자동 구매 희석**: 한국은 자동 구매 비중이 높아(과반으로
  알려짐) 조합별 인기 편차가 크게 압축된다. 관측된 번호별 효과가
  문헌(영·미 수동 선택 데이터, 수 배 차이)보다 훨씬 작은(±5~10%)
  이유다. 관측치는 "전체 구매 분포"이며 수동 편향은 그 일부다.
- 확실히 살아남는 신호는 **등차수열 패턴 과선택**과 **1~12 과선택**뿐.
- 유명 조합(1,2,3,4,5,6 등)은 당첨된 적이 없어 이 데이터로는 추정
  불가 → 문헌 기반 가중치를 유지(수천 배 과선택은 이론상 견고).

## 3. 모델 변경 (`popularity.py`)

1. **NUMBER_POPULARITY**: 문헌 기반 사전값(prior)과 실측 구간 테이블
   (1~12: 1.044, 13~31: 0.975, 32~45: 0.994, 7 ×1.03)의 기하 블렌드
   `w = prior^0.15 × empirical^0.85`. 사전값의 서열(7 최고, 4<5 등)은
   유지하되 크기를 실측 스케일로 대폭 축소.
2. **_COMPONENT_WEIGHTS** 재배분 (NNLS 회귀 기반):
   arithmetic 0.12→**0.42**, number_bias 0.34→0.33, grid_line 0.10→0.03,
   grid_cluster 0.06→0.01, famous 0.10→0.08(데이터로 추정 불가, 문헌 유지),
   pretty 0.10→0.06, birthday_combo 0.08→0.02, long_run 0.06→0.02,
   prev_overlap 0.04→0.03.
3. 공개 API 불변 (함수 시그니처/모듈 상수 이름 동일).
4. `config/default.yaml` model_version: be-v1.2.0-ev → **be-v1.3.0-ev**.

## 4. 보정 전/후 상관 (모델 점수 vs 실측 log popularity ratio)

| 모델 | Pearson | Spearman |
|---|---|---|
| 보정 전 (be-v1.2.0-ev) | 0.041 | 0.043 |
| 보정 후 (be-v1.3.0-ev) | **0.124** | **0.101** |
| 참고: 무제약 NNLS(성분 3개만) | 0.191 (5-fold CV 0.166) | 0.181 |
| 참고: 회차별 번호 테이블 자유 적합 | 0.277 (CV 0.148 — 과적합) | 0.276 |

- 배포 모델(0.124)은 무제약 최적(CV 0.166)보다 낮다. 이는 기존 테스트가
  보증하는 서열 구조(문헌 기반 사전값)를 보존하는 제약 하의 결과로,
  과적합 위험 대비 안정성을 택한 의도적 절충이다.
- 절대 상관이 낮은 것 자체가 정직한 결론이다: 자동 구매 희석 + 포아송
  잡음(평균 expected ≈ 8.9명) 하에서 조합 피처로 설명 가능한 분산은
  원래 작다.

## 5. 한계 (정직한 서술)

- **자동 구매 희석**: 수동 편향의 절대 크기는 식별 불가(수동 비율을
  모름). 추정된 계수는 "전체 구매 분포" 기준의 하한적 크기다.
- **포아송 잡음**: expected가 작은 회차는 잡음이 크다. expected 가중
  WLS와 +0.5 연속성 보정으로 완화했지만 제거는 불가.
- **유명 조합 미관측**: 극단적 인기 조합(1~6 등)은 당첨 표본에 없어
  실측 검증 불가 — 해당 성분은 문헌 신뢰에 의존한다.
- **시대 변화 미모형화**: 자동 구매 비율은 시간에 따라 변했을 것이나
  단일 계수로 통합 추정했다.
- 이 보정은 **당첨 확률과 무관**하다. 모든 조합의 1등 확률은 정확히
  1/8,145,060 로 동일하며, 개선되는 것은 상금 분할(공동 당첨자 수)
  기대치 추정뿐이다.
